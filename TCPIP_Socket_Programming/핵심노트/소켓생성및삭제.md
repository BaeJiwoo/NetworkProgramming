# 📚 네트워크 프로그래밍 마스터 가이드: Windows & Linux 상세 분석

이 문서는 윈도우(Winsock)와 리눅스(BSD) 소켓의 차이점 및 면접 대비를 위한 `socket()` 함수의 상세 옵션을 정리한 요약 노트입니다.

---

## 1. 플랫폼별 주요 차이점 (Windows vs Linux)

| 구분 | Windows (Winsock) | Linux (BSD Socket) |
| :--- | :--- | :--- |
| **헤더 파일** | `#include <winsock2.h>` | `#include <sys/socket.h>` |
| **라이브러리 초기화** | `WSAStartup()` 호출 필수 | 필요 없음 (바로 생성 가능) |
| **소켓 닫기** | `closesocket()` | `close()` |
| **라이브러리 종료** | `WSACleanup()` 호출 필수 | 필요 없음 |
| **소켓의 본질** | 별도의 객체 (`SOCKET` 타입) | 파일의 일종 (`int` 타입 / FD) |

---

## 2. 소켓 생성 함수 상세 분석 (`socket()`)

```cpp
SOCKET socket(int af, int type, int protocol);
```

### ① 주소 체계 (Address Family, `af`)

어떤 네트워크 프로토콜 체계를 사용하여 통신할지 결정합니다.

- **`AF_INET`**: IPv4 주소 체계 (32비트 주소). 가장 보편적으로 사용됨.
- **`AF_INET6`**: IPv6 주소 체계 (128비트 주소). 차세대 표준 주소 체계.
- **`AF_UNIX` / `AF_LOCAL`**: 동일한 호스트 내에서 프로세스 간 통신(IPC)을 할 때 사용.

### ② 소켓 타입 (Socket Type, `type`)

데이터가 전송되는 방식과 서비스의 특성을 결정합니다.

- **`SOCK_STREAM`**: 연결 지향형 (TCP)  
  - 데이터의 신뢰성 보장 (분실 시 재전송)  
  - 데이터의 순서 보장  
  - 바이트 스트림 방식 (데이터 경계 없음)

- **`SOCK_DGRAM`**: 비연결 지향형 (UDP)  
  - 전송 속도 중심 (신뢰성보다 성능)  
  - 데이터 유실 가능성 및 순서 역전 가능성 있음  
  - 메시지 기반 방식 (데이터 경계 존재)

### ③ 프로토콜 (`protocol`)

특정 주소 체계와 타입 내에서 사용할 세부 프로토콜을 지정합니다.

- **`0`**: OS가 주소 체계와 타입에 맞는 적절한 프로토콜 자동 선택
- **`IPPROTO_TCP`**: TCP 프로토콜 명시적 지정
- **`IPPROTO_UDP`**: UDP 프로토콜 명시적 지정

## 💻 통합 실전 코드 (Cross-Platform)

```cpp
#include <iostream>
#include <stdio.h>

#ifdef _WIN32
    #include <winsock2.h>
    #include <ws2tcpip.h>
    #pragma comment(lib, "ws2_32.lib")
    #define socket_close closesocket
#else
    #include <unistd.h>
    #include <sys/socket.h>
    #include <arpa/inet.h>
    #define socket_close close
    typedef int SOCKET;
    #define INVALID_SOCKET -1
#endif

int main() {

#ifdef _WIN32
    WSADATA wsa;
    if (WSAStartup(MAKEWORD(2, 2), &wsa) != 0) return 1;
    printf("[알림] 윈속 초기화 성공\n");
#endif

    SOCKET tcp_sock = socket(AF_INET, SOCK_STREAM, 0);
    SOCKET udp6_sock = socket(AF_INET6, SOCK_DGRAM, 0);

    if (tcp_sock == INVALID_SOCKET || udp6_sock == INVALID_SOCKET) {
        perror("소켓 생성 실패");
        return 1;
    }

    printf("[알림] IPv4 TCP 소켓 ID: %d 생성 성공\n", (int)tcp_sock);
    printf("[알림] IPv6 UDP 소켓 ID: %d 생성 성공\n", (int)udp6_sock);

    socket_close(tcp_sock);
    socket_close(udp6_sock);

#ifdef _WIN32
    WSACleanup();
#endif

    return 0;
}
```

## 3. 면접 예상 질문 & 답변 (Q&A)

### Q. `SOCK_STREAM`에서 데이터 경계가 없다는 의미는?

송신측에서 `send()`를 2번(각 100바이트) 호출해도 수신측은 `recv()` 1번으로 200바이트를 읽을 수 있습니다.  
데이터가 끊김 없는 흐름(Stream)으로 전송되기 때문이며, 애플리케이션 계층에서 길이 명시 또는 구분자를 통해 파싱해야 합니다.

### Q. IPv4와 IPv6를 동시에 지원하려면?

`AF_INET6` 소켓 생성 후 `setsockopt()`로 `IPV6_V6ONLY` 옵션을 0으로 설정하면 하나의 소켓으로 IPv4/IPv6 모두 수용하는 듀얼 스택 서버 구현이 가능합니다.

### Q. Windows에서 `WSAStartup` 없이 소켓 사용하면?

함수가 실패하며 `WSAGetLastError()`로 `WSANOTINITIALISED` 에러 반환.
