# 🌐 소켓 프로그래밍 학습 회고: 통신의 기초부터 고성능 모델까지

## 1. 학습 개요
이번 소켓 프로그래밍 학습의 핵심은 **'소켓 파일을 어떻게 설정하는가'**와 **'소켓을 통해 발생하는 입출력(I/O)을 어떤 방식으로 처리하는가'**에 대한 이해였습니다. 단순히 데이터를 주고받는 것을 넘어, 시스템 자원을 효율적으로 관리하기 위한 다양한 모델을 경험할 수 있었습니다.

## 2. 소켓 모델의 단계적 확장 (Build-up)
학습 과정은 단순한 형태에서 시작해 고성능 서버 아키텍처로 나아가는 체계적인 빌드업 과정을 거쳤습니다.

### ① 1:1 단일 통신 (Iterative Model)
- 가장 기초적인 단계로, 서버와 클라이언트가 1:1로 연결되어 데이터를 주고받는 구조입니다.
- 통신의 기본 흐름(Listen, Accept, Send, Recv)을 파악하기에 좋으나, 다중 접속 처리가 불가능하다는 한계를 확인했습니다.

### ② 멀티스레드 기반 모델 (Thread-per-Connection)
- 클라이언트의 접속마다 스레드를 새롭게 할당하는 방식입니다.
- 동시에 여러 클라이언트를 처리할 수 있지만, 접속자 수가 늘어날수록 스레드 생성 비용과 컨텍스트 스위칭(Context Switching) 오버헤드가 발생함을 배웠습니다.

### ③ IOCP (Input/Output Completion Port)
- 윈도우 환경에서 제공하는 비동기 I/O 완료 통지 메커니즘을 학습했습니다.
- I/O 작업을 커널에 맡기고 완료된 결과만 가져와 처리함으로써, 최소한의 스레드로 수만 개의 동시 접속을 효율적으로 처리하는 고성능 서버의 핵심 원리를 이해했습니다.

## 3. 핵심 인사이트 및 결론

네트워크 프로그래밍을 깊이 있게 공부하기 위해서는 단순히 API 사용법을 익히는 것보다 다음과 같은 요소들이 중요함을 깨달았습니다.

1.  **빌드업 과정의 이해**: 왜 더 복잡한 모델(IOCP 등)이 등장하게 되었는지, 이전 모델의 한계점이 무엇인지 파악하는 것이 우선되어야 합니다.
2.  **각 모델의 특징 분석**: 블로킹/논블로킹, 동기/비동기 방식에 따른 성능 차이와 적합한 사용 환경을 구분할 줄 알아야 합니다.
3.  **자원 관리의 효율성**: 소켓 설정과 입출력 처리 방식이 전체 시스템의 성능과 확장성에 직결됨을 인지하고 설계해야 합니다.
